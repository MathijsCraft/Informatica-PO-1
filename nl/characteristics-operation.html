Eigenschappen

Een ingebed systeem is een stuk hardware met software geïnstalleerd dat verschillende doeleinden kan hebben. Een ingebed systeem word vaak gebruikt als een controller van sommige functies in een groot systeem, een voorbeeld hiervan is bijvoorbeeld onze infrastructuur. Ingebedde systemen bestaan uit verschillende onderdelen en hebben altijd een geheugen en minstens één manier om data te ontvangen. De ingebedde systemen kunnen dingen meten, ze kunnen rekenen, signalen afgeven en nog veel meer. Er word heel vaak ook gebruik gemaakt van AI om het systeem te leren hoe het moet reageren op bepaalde data. Ingebedde systemen komen voor in vele systemen, ze zorgen dat de systemen intelligent gedrag kunnen vertonen, meer hierover kunt u lezen op de pagina 'Het gebruik van Embedded systemen' Je kan hardware en software vergelijken met bijvoorbeeld het menselijk lichaam, je kan het als volgt zien, de hardware is eigenlijk het lichaam en alle organen, alle fysieke elementen die ervoor zorgen dat een systeem werkt. De software kan je ook wel zien als het brein, zonder de software heb je niks aan de hardware en andersom geld hetzelfde. De software is wat een apparaat intelligent maakt, programmeurs werken hier al jaren aan om onze apparaten zo slim mogelijk te maken. Maar dat is nog niet alles, onder software valt bijvoorbeeld ook de programma's op uw computer die verschillende taken uitvoeren en dus verschillende doeleinden hebben. De date die hiervoor nodig is behoort ook tot de sectie software. Het samenvoegen van deze twee elementen resulteert in een ingebed systeem, zonder deze 2 elementen zouden we nooit de systemen kunnen maken die we nu hebben, elk jaar bedenken we steeds innovatievere oplossingen voor alledaagse problemen met behulp van ingebedde systemen. En doordat we gebruik maken van een combinatie van elementen is het mogelijk om specifieke functies binnenin een apparaat aan te passen en/of te toevoegen.

Embedded Systemen 101

101

Een ingebed systeem begint allemaal met hardware en software, maar wat houd dat nou precies in? Wat valt er onder hardware, en wat voor soort hardware hebben we het dan over? Uiteindelijk draait het erom dat de hardware de software voorziet van heel veel data, de software heeft deze data nodig om berekeningen te maken en daarmee dingen te voorspellen, controleren of vergelijken. We kunnen deze data verkrijgen door bijvoorbeeld een sensor te plaatsen. Een sensor verricht een meting en stuurt deze data door naar de software, dit word gedaan doormiddel van een signaal, vaak is dit een elektrisch signaal over bijvoorbeeld het moederbord van het systeem, het moederbord is een PCB (Printed Circuit Board) en heeft een heel circuit aan paden voor het versturen van signalen. Deze signalen kunnen worden opgevangen en daarna verwerkt door een computer of een speciaal ontwikkelde ontvanger. Nu we de meting van de sensor hebben omgezet in een elektrisch signaal, het signaal hebben verwerkt en de resultaten hebben weergegeven gaat het door het laatste stukje heen, een interface, een interface zet het elektrisch signaal om in binaire code. Computers maken gebruik van binaire code en dit bestaat uit series van enen en nullen. Als dit is gedaan gaat de software aan de slag. De software is zo gemaakt om deze code om te zetten in data die de gebruiker begrijpt. Een set van miljoenen enen en nullen is namelijk erg lastig om te begrijpen voor de gebruiker. Het systeem kan nu de data grafisch weergeven in bijvoorbeeld een grafiek of dergelijke. De data word vervolgens verbonden aan andere datalijnen, het vergelijken van data en het analyseren van data is dan mogelijk. De uitkomst hiervan varieert per situatie.
